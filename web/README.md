# Web Dashboard

This folder contains the static frontend dashboard for viewing scan results generated by the analyzer.

The UI is served by `server/server.js` and reads report data from API endpoints backed by `reports/report.json`.

## Purpose

- Display code vulnerabilities grouped by file
- Display dependency vulnerabilities from `npm audit`
- Show high-level security metrics (files, totals, critical/high counts)

## Runtime Flow

1. `npm run analyze` generates `reports/report.json`.
2. `npm start` serves this `web/` folder and API routes.
3. `web/app.js` fetches `GET /report` and renders cards/summary.

## Prerequisites

- Node.js 18+
- Existing report file at `reports/report.json` (run analysis first)

## Run

From repository root:

```bash
npm install
npm run analyze
npm start
```

Open:

- `http://localhost:3456/`

## API Endpoints Used by the Dashboard

- `GET /report` -> full report payload
- `GET /summary` -> summary-only payload
- `GET /issues` -> flattened issue list
- `GET /api/report` -> backward-compatible full report endpoint

## Expected Report Shape

The dashboard expects:

- `codeIssues` (array of files with `findings`)
- `dependencyIssues` (array of package vulnerability objects)
- `summary` (totals by severity)

## Dependency Issue Rendering Notes

Dependency cards are rendered from `dependencyIssues[*].via` entries.
Each vulnerability entry should include useful fields like:

- `title`
- `severity`
- `url` (for external advisory detail link)

## Troubleshooting

- If UI shows `Unable to load report`, run `npm run analyze` first.
- If API returns `404 Report not found`, confirm `reports/report.json` exists.
- If cards are empty, verify the JSON keys above are present in report payload.
- If server port is busy, set `PORT` and restart, for example: `PORT=4000 npm start`.
